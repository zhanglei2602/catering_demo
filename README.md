# 云汀私宴 Demo

这是一个围绕“私房菜饭店”场景制作的前后端一体 demo，包含两个页面：

- 菜单定制页：根据人数、预算、场景、偏好和忌口生成一套私房菜菜单
- 后厨调度页：模拟厨房持续运转、手动加单和多桌均衡出餐

项目使用：

- Python 提供页面服务和菜单生成接口
- 原生 HTML + CSS + JavaScript 做页面展示和交互

## 启动方式

在项目目录下运行：

```bash
python app.py
```

启动成功后，终端会输出本地访问地址。

默认地址：

```text
http://127.0.0.1:8000
```

## 页面说明

### 1. 菜单定制页

地址：

```text
http://127.0.0.1:8000/
```

作用：

- 输入人数、用餐场景、预算风格、口味偏好、忌口和补充备注
- 生成一套更贴近真实私房菜节奏的菜单
- 展示每道菜的介绍、推荐理由、上桌表达、整桌预算和主厨备注

适合展示：

- 前台菜单推荐
- 客户点餐体验
- “根据条件生成菜单”的场景演示

### 2. 后厨实时调度页

地址：

```text
http://127.0.0.1:8000/kitchen
```

作用：

- 配置厨房资源，例如厨师人数、冷菜台、热菜灶、炖煮炉、蒸箱
- 厨房持续运转，时间会一直推进
- 手动点击“加入随机一桌”，在当前时刻生成一桌新菜单并进入后厨流程
- 模拟多桌并行时的出餐调度，尽量避免只让一桌连续上满、其他桌长时间无菜可上

页面主要区域：

- `03 / 后厨工位`：看当前各个工位正在处理什么菜
- `04 / 前场桌面`：只显示当前仍在等待或持续出餐中的桌台
- `05 / 出餐流`：最近刚送到客人桌上的菜
- `06 / 桌台详情`：看每桌当前进度、等待情况和菜品状态
- `07 / 出餐记录`：已经完整出齐的桌台会移到这里，并记录本桌出了哪些菜

适合展示：

- 后厨调度
- 多桌并发出餐
- 实时运营看板
- 餐厅现场感演示

## 目前实现的能力

- 内置一套模拟私房菜菜库，覆盖前菜、汤品、海鲜、禽类、肉菜、时蔬、主食、甜品
- 菜单生成会综合人数、预算、偏好、忌口和场景来选菜
- 后厨调度会区分冷菜、热菜、炖煮、蒸制等不同工位和耗时
- 厨房会在资源限制下持续调度，尽量兼顾各桌等待体验
- 页面支持动态展示，能看到时间推进、工位变化、前场桌面变化和出餐记录

```text
static/kitchen.css        后厨实时调度页样式
static/kitchen.js         后厨实时调度页脚本
```

## 备注

- 这是一个本地演示项目，不依赖数据库
- 当前菜单和调度数据为模拟数据，适合做交互和方案展示
- 如果后续要接入真实业务，可以把菜单数据、订单来源和后厨状态接入真实系统
